### A3 — LeatherCleaner Strong silikon (`conv_01KQQZ4ZJW8EVTPG2PXRMHA32Q`)

```
1. searchProducts({brand: GYEON, exactMatch: "LeatherCleaner Strong", query: "GYEON LeatherCleaner Strong",
                   templateGroup: interior_cleaner, templateSubType: leather_cleaner})
2. getProductDetails({sku: Q2M-LCSYA1000M})
3. searchFaq({query: "silikon içerir mi silikonsuz mu", sku: Q2M-LCSYA1000M, limit: 5})
```

**Policy uygulaması mükemmel:**
- Madde 1: lastFocusSku VAR (Q2M-LCSYA1000M) → getProductDetails çağrıldı (faqs alanı kontrol edildi)
- Madde 1b: detail.faqs'ta "silikon" topic'i eşleşmedi (DB'de bu topic'te FAQ yok)
- Fallback: searchFaq+sku tetiklendi → SKU-filtered FAQ aramada doğru cevap bulundu
- Madde 3: SKU filter korunduğu için yanlış ürün cevabı gelmedi

Phase 1.1.13K leather taxonomy doğru: `interior_cleaner / leather_cleaner` filter aktif (eskinin `leather_care` enum'undan kaldırılmıştı).

### A4 — silikonsuz şampuan (`conv_01KQQZ3BPVKPQGDR18W4AA8TNS`)

```
1. searchProducts({query: "silikonsuz şampuan", templateGroup: car_shampoo,
```

Not: Bu davranış FAQ policy patch'i ile alakasız olabilir (bot LLM stochastic karar) ama 14D-mini coverage audit'te slot extractor "silikonsuz" pattern'ini doğrulamak gerekecek (eğer slot extractor varsa otomatik filter zaten ekleniyor olabilir).
