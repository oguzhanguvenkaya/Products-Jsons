#### Conv 6 — Gommanera Superlux: Kategori Halüsinasyonu

**Soru:** "Gommanera Superlux 5 lt mevcut mu?"
**Bot cevap:** "**FRA-BER FRA-BER Superlux Suya Dayanıklı Lastik Parlatıcı** ürününü buldum"

**Hata:**
- "Gommanera Superlux" DB'de **abrasive_polish (katı pasta)**, lastik parlatıcı DEĞİL
- Bot exactMatch="Gommanera Superlux" KULLANMADI, sadece "Superlux" semantic match → yanlış FRA-BER ürünü
- "FRA-BER FRA-BER" duplicate marka adı (formatting bug)
- Plan §14B.7 ürün adı uydurma yasağı **kapsam dışı** (gerçek ürün gösterdi) ama **kategori halüsinasyonu** ile yasak ihlali

**Root cause:** searchProducts.exactMatch description (14C.2'de kısaltıldı) yetersiz — bot multi-word model adı için exactMatch zorlanmıyor.

## Kritik Bulgular — Düzeltilmesi Gereken 5 Madde

### R1 — "Kataloğda kaç ürün?" — Catalog count cevap kuralı eksik

**Mevcut:** Bot searchProducts result count'unu yorumlayıp "1 ürün" diyor (yanlış)

**Düzeltme önerisi (instruction +3 satır):**
```
"Kataloğda kaç ürün?" / "kaç farklı ürün var?" / "marka sayısı?" sorularında:
- ASLA searchProducts result count kullanma — bu sadece o sorguya dönen ürün
- "MTS Kimya katalogunda yüzlerce ürün ve varyant bulunmaktadır. Spesifik
  bir kategori veya marka için arama yapayım" gibi sayısız + arama önerisi
- Tam katalog için satıcıya yönlendir
```

### R2 — exactMatch zorlama (multi-word model adı)

**Mevcut:** "Gommanera Superlux" gibi 2-3 kelimelik model adı için bot exactMatch kullanmıyor

**Düzeltme önerisi (search-products.ts exactMatch description'a +1 satır):**
```
ÖZELLIKLE: kullanıcı 2-3 kelimelik tam model adı yazdıysa
(örn. "Gommanera Superlux", "Bathe+ Plus") exactMatch ZORUNLU —
yoksa wrong-brand semantic match riski (Gommanera Superlux ≠ FRA-BER Superlux).
```

### R3 — silikonsuz şampuan metaFilter çağrısı zayıfladı

**Mevcut:** 14C.4 metaFilters tool desc kısaltma sonrası bot "silikonsuz" → metaFilter[silicone_free=true] çağırmıyor

**Önceki (baseline 14B-pre.3 A/B):** Bot kendiliğinden çağırıyordu (beklenmedik kazanım idi)
**Şimdi:** "filtre bulamadım" generic cevap

**Düzeltme önerisi:** metaFilters description'a 1 satır vurgu ekle veya silikonsuz örneği daha belirgin yap

### R4 — Multi-turn state retention bug (Conv 15)

**Mevcut:** "Kurulama havlusu öner" sonra "En çok emen hangisi?" → alakasız cevap

**Tahmin:** lastProducts state retention'ı genel sorularda doğru çalışmıyor olabilir; veya bot LLM bağlamı LLM context'inden düşürmüş

**Düzeltme önerisi:** Daha geniş test gerek — bu commit kapsamı dışında, ayrı debugging
