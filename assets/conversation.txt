     🔐 VERİ TOPLAMA STRATEJİSİ

     KRİTİK KURAL: URL Scraping ZORUNLU!

     ⚠️ URL'i olan TÜM ürünlerin sayfaları kazınacak.
     ⚠️ Kazınan verinin DOĞRU ÜRÜN olduğu doğrulanacak.
     ⚠️ Sadece doğrulanmış veriler kullanılacak.

     Veri Kaynakları (Hepsi Kullanılacak)

     1. all_categories.json → content.full_description (Türkçe)
     2. URLs_merged.csv → Product Description sütunları
     3. URL Scraping → Tüm URL'ler kazınacak (ZORUNLU!)

     Üç kaynaktan gelen veri BİRLEŞTİRİLECEK.

     URL Scraping ve Doğrulama Süreci

     ┌─────────────────────────────────────────────────────────────────┐
     │  HER ÜRÜN İÇİN (URL'si varsa):                                  │
     │                                                                  │
     │  1. URLs_merged.csv'den URL listesini al                        │
     │  2. Her URL için WebFetch çağır                                 │
     │  3. Sayfa içeriğini analiz et                                   │
     │  4. DOĞRULAMA: Bu sayfa gerçekten bu ürün mü?                   │
     │     - Marka eşleşmesi kontrol                                   │
     │     - Ürün adı kelimeleri kontrol                               │
     │     - SKU/Barkod kontrolü                                       │
     │  5. Doğrulama skoru >= 0.7 → KABUL                              │
     │  6. Kabul edilen sayfalardan veri çıkar                         │
     │  7. Tüm kaynakları birleştir                                    │
     └─────────────────────────────────────────────────────────────────┘

     Doğrulama Algoritması (BARCODE BAZLI)

     ⚠️ SKU ile doğrulama YAPILMAZ (SKU'lar firmaya özel)
     ⚠️ BARCODE kullanılmalı (evrensel ürün kodu)

     BARCODE Kaynağı: Products_with_barcode.csv
        StokKodu (SKU) → Barkodu (BARCODE) eşleşmesi

     Doğrulama Puanı = (Marka × 0.3) + (Ürün Adı × 0.4) + (BARCODE × 0.3)

     Marka Kontrolü (0.3 puan):
     - "GYEON", "Q2M", "Q2" → GYEON ürünleri
     - "Menzerna", "MENZERNA" → Menzerna ürünleri
     - "FRA-BER", "Fra-Ber" → Fra-Ber ürünleri

     Ürün Adı Kontrolü (0.4 puan):
     - Title'dan anahtar kelimeler çıkar: "SilkDryer", "EVO", "70x90"
     - Sayfada bu kelimeler var mı?

     BARCODE Kontrolü (0.3 puan):
     - Products_with_barcode.csv'den Barkodu al
     - Sayfada bu barcode var mı?
     - Örnek: "8809432671480" (GYEON SilkDryer)

     Eşik: score >= 0.7 → KABUL, < 0.7 → REDDET

     Reddedilen URL'ler

     Reddedilen URL'ler loglanacak:
     - rejected_urls.json
     - Neden reddedildi
     - Hangi ürün için
     - Manuel kontrol için işaretlenir

     Agent 0: URL Scraper & Verifier (YENİ - İLK ÇALIŞACAK)

     # URL SCRAPER & VERIFIER AGENT

     ## Görevin
     URL'i olan ürünler için TÜM URL'leri kazı ve doğru ürün olduğunu doğrula.

     ⚠️ BU AGENT ZORUNLU - URL'i olan her ürün için çalışmalı!

     ## Input
     - sku: "{SKU}"
     - title: "{PRODUCT_TITLE}"
     - brand: "{BRAND}"
     - urls: ["{URL1}", "{URL2}", "{URL3}", "{URL4}", "{URL5}"]

     ## Görev Adımları

     ### 1. Her URL için WebFetch Çağır
     for each url in urls:
         content = WebFetch(url)
         process(content)

     ### 2. DOĞRULAMA (KRİTİK!)
     Her sayfa için doğrulama yap:

     A) Marka Kontrolü (0.3 puan):
        - "{BRAND}" veya varyasyonları sayfada var mı?
        - GYEON: "GYEON", "Q2M", "Q2", "Gyeon Quartz"
        - MENZERNA: "Menzerna", "MENZERNA"
        - FRA-BER: "FRA-BER", "Fra-Ber", "Fraber"

     B) Ürün Adı Kontrolü (0.4 puan):
        - Title'dan anahtar kelimeler çıkar
        - Örnek: "SilkDryer EVO 70x90" → ["SilkDryer", "EVO", "70x90", "70×90"]
        - Bu kelimeler sayfada var mı?

     C) SKU Kontrolü (0.3 puan):
        - "{SKU}" sayfada var mı?
        - Varyasyonlar: Q2M-SDE7090C, Q2MSDE7090C, SDE7090C

     D) Puan Hesapla:
        - score = (marka × 0.3) + (ürün_adı × 0.4) + (sku × 0.3)
        - score >= 0.7 → KABUL
        - score < 0.7 → REDDET

     ### 3. Veri Çıkarma (Sadece KABUL Edilen URL'lerden)
     - Teknik özellikler (specs)
     - Kullanım talimatları
     - FAQ (varsa)
     - İlişkili ürünler
     - Görseller (URL'leri)

     ## Görevin
     Bir kategori için tüm ürünlerin açıklamalarını analiz et ve
     eklenebilecek yeni specs alanlarını tespit et.

     ## Input
     - category: "{CATEGORY_NAME}"
     - category_products: [{sku, enriched_description}, ...]
     - current_specs_schema: {mevcut specs alanları}

     ## Görev

     ### 1. Tüm Ürün Açıklamalarını Tara
     Her üründe tekrar eden teknik özellik kalıplarını bul:
     - "VOC içermez" → voc_free: boolean
     - "P1200 kum" → grit_removal: string
     - "30 dakika 110°C" → temperature_resistance: number

     ### 2. Sıklık Analizi
     Her bulunan özelliğin kaç üründe geçtiğini say.
     %50'den fazla üründe geçiyorsa → standart alan olabilir.

     ### 3. Öneri Oluştur
     Yeni alan önerileri:
     - Alan adı (snake_case)
     - Veri tipi (string, number, boolean, array)
     - Açıklama
     - Kaç üründe tespit edildi

     ## Output Format (JSON)
     {
       "category": "abrasive_polishes",
       "analysis_summary": {
         "products_analyzed": 40,
         "new_fields_suggested": 3
       },
       "suggested_fields": [
         {
           "field_name": "voc_free",
           "data_type": "boolean",
           "description": "VOC (Uçucu Organik Bileşik) içermez",
           "detected_in_products": 15,
           "percentage": 37.5,
           "sample_values": [true, false]
         },
         {
           "field_name": "grit_removal",
           "data_type": "string",
           "description": "Giderilebilen minimum zımpara kumu",
           "detected_in_products": 28,
           "percentage": 70,
           "sample_values": ["P1200", "P1500", "P2000"]
         }
       ],
       "recommendation": "grit_removal alanı %70 üründe tespit edildi, eklenmesi
     önerilir."
     }
